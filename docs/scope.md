# Project Scope Statement  

## Project Name:
Beginner Workout Guide Website  

## Project Purpose Feedback Loops  

**Why are we doing this project?**  
The purpose of this project is to create a simple website that provides beginners with a clear and structured workout routine. The website will focus on helping users understand how to organize a weekly workout schedule and follow basic exercises. This project will also demonstrate skills in web development, documentation, and project management.

**Who needs to provide feedback?**  
- Instructor  
- Project Manager through self-review  

**When and how?**  
Feedback will be collected after completing the scope document, during development, and before final submission through instructor review and self-evaluation.

## Description People Resources  

**How would you describe this project?**  
This project involves building a two-page website using HTML and CSS that is hosted on GitHub Pages. The homepage will introduce the purpose of the website and explain the importance of consistency in fitness. The second page will include a structured weekly workout schedule (e.g., upper body, lower body, and rest days), a list of beginner exercises such as push-ups, squats, and rows, and a section with basic fitness tips like proper rest and hydration. The website will use a single external CSS file to maintain consistent styling across both pages.

**Who are the key team members?**  
- Carlos Gomez 
- Instructor: Evaluator  

**What resources are needed?**  
- Computer and internet access  
- Markdown and editor app
- GitHub for repository and hosting  
- Basic knowledge of HTML and CSS  


## Desired Results Priorities  

**What specific outcomes must we achieve to be successful?**  
- A two-page website with clear navigation between pages  
- A homepage with the introduction and purpose  
- A second page with a weekly workout schedule and exercise list  
- Consistent styling applied through one CSS file  
- A properly organized GitHub repository with required documentation  

**How would you prioritize these constraints? (1 = most important)**  
- Time: 1  
- Quality: 2  
- Budget: 3  

## Time  

**When is the project due?**  
The project must be completed and submitted by May 10, 2026.

## Quality  

**What does done look like? How will we know?**  
The project is complete when both pages are fully functional, properly linked, visually consistent, and include all required content such as the workout schedule, exercise list, and fitness tips. All files must be correctly organized in the GitHub repository.

## Budget  

**What is the budget?**  
The budget is $0 since only free tools such as GitHub and a code editor will be used.

## Exclusions Acceptance Criteria  

**What is out of the project's scope and should NOT be included?**  
- No JavaScript functionality  
- No backend or database  
- No user accounts or login system   

**Who needs to sign off on the project and when?**  
The instructor will review and approve the project upon final submission.
