# Project Retrospective

## What Went Well

One thing that went well during this project was creating a clear structure before building the website. Having the scope statement and project plan helped make the work feel more organized. The website topic was also simple enough to manage, which made it easier to focus on the HTML, CSS, and documentation requirements, overall very organized. 

## What Went Wrong

One challenge was making sure all files were organized correctly in the GitHub repository. Since the project required website files in the root directory and documentation inside the folder, it was important to check the file structure carefully overall being knowlegdable on navigating Git Hub in general. Another challenge was keeping the project simple and not adding unnecessary features. The biggest issue that went wrong was having a field training exercise that had me with no technology for 5 days. 

## What I Would Do Differently

Next time, I would start the documentation earlier and test the website after each major change. I would also use a checklist from the beginning to make sure each file and requirement is completed on time. This would help prevent last minute mistakes and make the final submission process smoother.
